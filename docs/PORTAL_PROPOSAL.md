## 1. エグゼクティブサマリー

**Engineer Skill Lab** は、エンジニアが実務で必要とするスキルを「ゲーム感覚で楽しく鍛える」ことに特化した学習アプリポータルサイトです。

クラウド設計、DevOps、セキュリティ、ネットワーク、CS基礎など9カテゴリ・全15本のインタラクティブアプリを収録。ドラッグ&ドロップ型パズル、RPG形式のクエスト、リアルタイム対戦など多様な形式で、テキストや動画では得られない「手を動かす」学習体験を提供します。

第1弾として開発完了済みの「クラウドアーキテクチャパズル」を皮切りに、約12ヶ月で全15アプリを段階的にリリースし、エンジニア学習プラットフォームとしての地位を確立します。

### 主要KPI目標

| 指標 | 6ヶ月目標 | 12ヶ月目標 | 24ヶ月目標 |
|------|-----------|------------|------------|
| 月間アクティブユーザー | 10,000 MAU | 50,000 MAU | 200,000 MAU |
| 収録アプリ数 | 6本 | 12本 | 20本+ |
| 平均セッション時間 | 10分 | 15分 | 20分 |
| 月間再訪率 | 40% | 55% | 65% |
| NPS（推奨度） | +30 | +45 | +55 |

## 2. 企画背景と課題

### 2.1 市場背景

エンジニア人材の需要は世界的に拡大を続けており、特にクラウド・DevOps・セキュリティ領域のスキルギャップは年々深刻化しています。リスキリング・アップスキリングの需要は高まる一方、既存の学習手段には構造的な課題があります。

### 2.2 既存の学習手段の課題

| 課題 | 詳細 |
|------|------|
| **学習の断片化** | スキル領域ごとに別のプラットフォームを使い分ける必要があり、学習の一貫性が保てない |
| **受動的な学習** | 動画視聴やテキスト読解が中心で、手を動かす実践機会が乏しい |
| **高い学習コスト** | ハンズオンラボはクラウド利用料が発生し、失敗のコストが学習の障壁になる |
| **モチベーションの欠如** | 達成感やフィードバックが少なく、学習の継続率が低い |
| **スキルの可視化が困難** | 何をどの程度習得したか、体系的に把握する手段がない |

### 2.3 本企画の解決アプローチ

- **ワンストップポータル：** 9カテゴリのスキルを1つのプラットフォームで横断的に学習
- **ゲーミフィケーション：** パズル・RPG・対戦・シミュレーションなど多様な形式で能動的な学習を実現
- **ゼロコスト実践：** ブラウザ上で完結し、クラウド利用料やセットアップが一切不要
- **即時フィードバック：** スコア・ランク・模範解答により、学習効果をリアルタイムで実感
- **スキルマップ：** カテゴリ横断の進捗可視化で、自分の強みと弱みを一目で把握

## 3. プロダクト概要

### 3.1 プロダクト構成

```
Engineer Skill Lab（ポータルサイト）
├── ポータル機能（検索・フィルタ・ソート・結果シェア/動画導線）
├── スキルマップ（進捗可視化・レーダーチャート）
└── 学習アプリ群（全15本、段階的にリリース）
    ├── ☁️ クラウド（3本）
    ├── 🔄 DevOps（3本）
    ├── ⚙️ バックエンド（1本）
    ├── 🔒 セキュリティ（1本）
    ├── 🌐 ネットワーク（1本）
    ├── 📚 CS基礎（3本）
    ├── 🖥️ インフラ（2本）
    └── 🗄️ データベース（1本）
```

### 3.2 ポータルサイトの役割

ポータルサイトは単なるアプリ一覧ではなく、以下の価値を提供する「学習ハブ」として機能します。

- **発見：** カテゴリ・難易度・タグによる最適なアプリの発見
- **動機付け：** ランキング・バッジ・ストリークによる継続学習の動機
- **体系化：** スキルマップによる学習進捗の体系的な可視化
- **コミュニティ：** ユーザー同士の学び合いと情報共有

```

└── src/apps/<appId>/
    ├── App.tsx                    ← アプリ本体（ドラッグ&ドロップ/CTF等）
    ├── components/               ← UI部品（タイル、入力、結果画面など）
    ├── content/

    │   ├── learning.json        ← ヒント・解説・模範解答
    │   └── youtube.json         ← 結果→解説動画（動画ID/章立て）
    └── youtube-entrypoint.ts    ← 結果画面から埋め込み/遷移するための薄い集約層
```

## 9. 各アプリの詳細仕様

### 9.1 クラウドアーキテクチャパズル

- **形式：** ドラッグ&ドロップ型パズル
- **内容：** クラウドでよく使われる役割のコンポーネントを配置・接続してアーキテクチャ構成図を完成（ベンダー非依存の一般的な考え方）
- **シナリオ：** 3問 × 3クラウド = 9パターン（★〜★★★）
- **採点：** 110点満点、S〜Dランク判定、層別スコア

### 9.2 SQL道場

- **形式：** クエリ対戦パズル
- **内容：** テーブルと期待結果を提示し、正しいSQLクエリを構築。実行結果をリアルタイムプレビュー
- **段階：** SELECT基礎 → JOIN → サブクエリ → ウィンドウ関数 → パフォーマンスチューニング
- **特徴：** ブラウザ内SQLエンジン（sql.js）で即座に実行・検証

### 9.3 Gitブランチクエスト

- **形式：** ターミナル入力RPG（コマンド入力→正誤判定→進行）
- **内容：** 各ステップで `git` の“実行コマンド（またはコマンド順序）”を入力。正しい入力に応じてブランチツリーと疑似ログが更新されます。
- **段階：** ブランチ作成（例: `git checkout -b feature`）→ マージ（`git merge feature`）→ コンフリクト解決（oursを採用→ステージング）→ チェリーピック（例: `git cherry-pick <hash>`）→ Git Flow（例: `git flow release finish v1.0.0`）→ リベース（`git rebase main`）
- **特徴：** コンフリクトマーカー表示、チェリーピック候補コミットの提示、実行コマンドが行動ログに残る（学習用の“手順ログ”として機能）

### 9.4 Dockerコンテナラボ

- **形式：** シミュレーション
- **内容：** Dockerfileの命令をブロック形式で組み立て、コンテナイメージを構築
- **段階：** 基本命令 → マルチステージビルド → Docker Compose → 最適化
- **特徴：** イメージサイズやレイヤー数がリアルタイムで変化し、最適化を体感

### 9.5 API設計ワークショップ

- **形式：** インタラクティブ設計ツール
- **内容：** REST/GraphQLのエンドポイントを設計し、リクエスト/レスポンスをプレビュー
- **段階：** CRUD設計 → バリデーション → ページネーション → バージョニング → GraphQL
- **特徴：** OpenAPIスペックの自動生成。設計のベストプラクティスをリアルタイムフィードバック

### 9.6 セキュリティ脆弱性ハンター

- **形式：** CTF（Capture The Flag）
- **内容：** 意図的に脆弱性が埋め込まれたコードから問題箇所を特定
- **段階：** XSS → SQLインジェクション → CSRF → 認証不備 → 総合問題
- **特徴：** OWASP Top 10を網羅。修正コードの提案も求められる実践的な内容

### 9.9 Linuxコマンドクエスト

- **形式：** ターミナルRPG
- **内容：** 仮想ターミナルで実際のコマンドを打って冒険を進行
- **段階：** ファイル操作 → パーミッション → プロセス管理 → シェルスクリプト → ネットワーク系
- **特徴：** コマンドの実行結果を仮想ファイルシステム上でシミュレート

### 9.10 CI/CDパイプラインビルダー

- **形式：** ブロック組立
- **内容：** ビルド→テスト→デプロイのブロックを接続してパイプラインを構築
- **段階：** 基本パイプライン → 並列実行 → 条件分岐 → マトリクスビルド → デプロイ戦略
- **特徴：** GitHub Actions / GitLab CI形式のYAMLプレビュー

### 9.11 ネットワークトポロジービルダー

- **形式：** シミュレーション
- **内容：** サブネット・ルーター・ファイアウォールを配置してネットワークを設計
- **段階：** LAN設計 → VLAN → ルーティング → ファイアウォールルール → 総合設計
- **特徴：** パケットの流れをアニメーションで可視化。障害注入テストも可能

### 9.12 Kubernetesオーケストレーター

- **形式：** シミュレーション
- **内容：** Pod, Service, Deployment, Ingressをビジュアルに配置してクラスタ構築
- **段階：** Pod管理 → ReplicaSet → Service → Ingress → HPA + 障害シミュレーション
- **特徴：** ノードの負荷状態やPodの状態遷移をリアルタイムで可視化

### 9.13 システムデザイン模擬面接

- **形式：** AI対話
- **内容：** Twitter, Uber等の設計問題に対し、コンポーネント図を描きながらAIがフィードバック
- **特徴：** Claude APIを活用した自然な面接シミュレーション

### 9.15 監視ダッシュボードシミュレーター

- **形式：** シミュレーション
- **内容：** 本番障害を模したシナリオで、メトリクス・ログ・トレースから原因を特定
- **特徴：** SRE訓練ツール。SLO/SLI/エラーバジェットの概念も組み込み

## 13. リスクと対策

| リスク | 影響度 | 対策 |
|--------|--------|------|
| アプリの品質担保が困難（15本の開発負荷） | 高 | Phase分けで段階リリース。各Phaseでユーザーテストを実施し、品質を担保してから次へ |
| 特定アプリの利用が偏る | 中 | おすすめ機能のアルゴリズム最適化。不人気アプリは改善 or 統合を検討 |
| 技術トレンドの変化でコンテンツが陳腐化 | 中 | データ駆動設計（サービス定義をJSON化）で迅速な更新。コミュニティ投稿で持続的なコンテンツ生成 |
| 導線CTR/視聴継続が想定を下回る | 高 | 結果画面の動画レコメンド・埋め込みUXを改善し、視聴完了率/リピートを上げる |
| 競合サービスの参入 | 中 | 15アプリの幅広さとゲーム性の高さが参入障壁。先行者優位でコミュニティを構築 |
| 開発リソースの不足 | 高 | コアチーム3名+フリーランス活用。アプリの共通基盤（採点・UI）をライブラリ化し開発効率向上 |

## 14. まとめ

**Engineer Skill Lab** は、「ゲームで楽しくスキルアップ」という明確なコンセプトのもと、エンジニア学習の在り方を変えるポータルサイトです。

第1弾「クラウドアーキテクチャパズル」は開発完了済みであり、ポータルサイトのMVPも実装が完了しています。Phase 1として即座にユーザーテストとフィードバック収集を開始し、12ヶ月で全15アプリの段階的リリースを目指します。

### 本企画の3つの強み

1. **ゲーム × 学習の融合** — 座学では得られない「手を動かす楽しさ」
2. **9カテゴリ・15アプリのワンストップ** — エンジニアスキルを網羅的にカバー
3. **ポップで可愛いUI** — 学習のハードルを徹底的に下げる親しみやすさ

これら3つの差別化要素により、個人学習者から企業研修まで幅広いニーズに応え、エンジニア学習プラットフォームとしての地位を確立します。
