## 1. リリース1の訴求とチャネル

**一言コピー（例）**

- 「DNS も LB も、**置いて、線でつなぐだけ**。クラウドの“つながり”が体でわかるブラウザパズル。」
- 「インフラ勉強、**まずは配線から**。無料・ログイン不要の **Engineer Skill Lab**。」

**チャネル役割**

| チャネル | 役割 |
|----------|------|
| **Shorts** | 0〜2 秒でフック → 画面で接続・採点が伝わる → 締めで CTA。チャレンジ名はテロップで必ず出す |
| **X** | 同じ動画へのリンク、**別角度の一文**、スレッドで「1 問＝1 ポスト」解説。ハッシュタグは **2 個まで**目安 |

**本番 URL**：撮影・投稿前に、実際に開ける URL を [README](../README.md) / デプロイ手順で確認し、概要欄・プロフィール・固定ポストに同じものを載せる。

## 2. YouTube Shorts — 撮影の型

台本に書いた **【画面】** に合わせて録画する。ナレーションは読み上げ or テロップのみでも可。

| 記号 | 意味 |
|------|------|
| **0〜2秒** | フック（テロップ大きく）。無音＋テロップだけでも可 |
| **【画面】** | キャンバス・カタログ・接続・採点のどれを映すか |
| **【テロップ】** | 画面上の文字案。短く切る |
| **（無音）** | BGM のみ。操作音入れてもよい |

縦動画（1080×1920）推奨。UI が小さいときは **該当パーツにズーム** してから接続シーンを撮る。

## 3. YouTube Shorts — 共通パート（毎本の締め）

**尺：約 8〜12 秒（前の本編のあとに必ず付ける）**

| 秒 | 【画面】 | ナレーション（例） | 【テロップ】 |
|----|----------|----------------------|----------------|
| 締め① | 採点結果 or S ランクが出た瞬間を 1 秒見せる → ポータル or アプリタイトル画面 | 「**Engineer Skill Lab** の **クラウドアーキテクチャパズル**。ブラウザで無料、全 8 問。」 | Engineer Skill Lab |
| 締め② | チャレンジ一覧が見えるなら一瞬スワイプ | 「プロフィールのリンクから今すぐ。いいねとチャンネル登録お願いします。」 | リンクは概要欄 |

※「チャンネル登録」は YouTube のみ。X では [§6](#6-x--文案テンプレ) の CTA に差し替える。

## 7. X — スレッド台本（例：チャレンジ1）

**1/n**

```
スレッド：「基本的なWebサイト」で何を見るか（1/6）

静的サイトの話。
最初に「**ユーザーが何を通ってページに届くか**」だけ追う。

※無料パズル Engineer Skill Lab
【本番URL】
```

**2/n**

```
（2/6）名前を IP に変える。だから **DNS**。
```

**3/n**

```
（3/6）静的ファイルは **CDN** で近くに置く。遅延と負荷を減らす。
```

**4/n**

```
（4/6）このチャレンジでは、**DB 不要**の前提が多い。
「いらない線を引かない」も設計。
```

**5/n**

```
（5/6）パズル上は「**正しい部品**」と「**正しい接続**」で採点。
抜けがあるとスコアが落ちる。
```

**6/n**

```
（6/6）次は「動的Webアプリ」で **API と DB** が出てくる。
まずチャレンジ1をクリアしてからがおすすめ。

【本番URL】
```

他チャレンジも同じ **6 ポスト構成**（導入 → キーワード2〜3本 → パズルでの学び → CTA）に差し替えて使う。
